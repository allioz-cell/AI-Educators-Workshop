Hypervisors

Where the hypervisor sits determines the performance and the use case.

Type 1 — bare metal

Hardware → hypervisor → virtual machines

The hypervisor is the operating system. It installs straight onto the hardware, so there is no host OS competing for resources. This is what runs production servers.

Examples: VMware ESXi, Microsoft Hyper-V Server, Proxmox VE, XCP-ng

Type 2 — hosted

Hardware → host OS → hypervisor → virtual machines

Runs as an application on top of Windows, macOS or Linux. Slower, because everything passes through the host OS, but perfect for a technician testing on a laptop.

Examples: VirtualBox, VMware Workstation, Parallels Desktop

Resource requirementsEvery virtual machine needs its own allocation of CPU cores, RAM and disk, plus a valid license for its guest operating system. Virtualization also requires hardware support enabled in firmware — Intel VT-x or AMD-V. A machine that refuses to start a 64-bit guest almost always has that setting turned off.

Containers are not virtual machines

Both provide isolation; only one carries an entire operating system with it.

Cloud characteristics and deployment models

Rapid elasticity, metered billing and tenancy are the three ideas that generate most cloud questions.

Characteristics the exam names

  • On-demand self-service: a user provisions resources through a portal without waiting on a person.
  • Rapid elasticity: capacity scales up during a spike and back down afterward.
  • Measured / metered service: you are billed for exactly what you consume.
  • Resource pooling: one physical infrastructure serves many tenants.
  • High availability: workloads run across zones so one failure does not take the service down.
  • File synchronization: changes propagate to every device signed into the account.

Deployment models

  • Public: a provider's shared infrastructure, open to any customer.
  • Private: infrastructure dedicated to one organization, on site or hosted. The answer whenever regulation demands sole tenancy.
  • Hybrid: private and public joined, so sensitive data stays in and bursty workloads go out.
  • Community: shared by several organizations with the same compliance needs, such as a group of hospitals.
Desktop virtualizationVDI puts the desktop itself in the data center; the endpoint becomes a thin client that streams the screen. It keeps data off the device entirely, which is why questions about lost laptops and contractor access tend to land here.
